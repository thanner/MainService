The process starts when the Resource 1 brings in the defective computer. Next, it receives the repair cost calculation. Subsequently, one of the 2 alternative procedures is executed. If "yes", the Resource 1 continues the process. If "no", the Resource 1 takes the computer home. In any case, the process ends. The process starts when the Resource 2 prepares a repair cost calculation. Next, one of the 2 alternative procedures is executed. The Resource 2 receives a message. In any case, the Resource 2 receives a message and then conducts the activity. The subprocess contains the following steps. One of the 2 alternative procedures is executed. In the first procedure, the Resource 2 checks and configures the software. In the second procedure, the Resource 2 checks and repairs the hardware. In any case, the subprocess ends. Subsequently, the Resource 2 tests the system funcionality. If it is required, the latter steps are repeated. In that case the Resource 2 continues with conducting the activity. After that, the process continues. In any case, the process ends.
